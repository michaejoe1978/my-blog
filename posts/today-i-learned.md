# 오늘 배운 것

클로드 코드와 함께 정적 블로그를 직접 만들어 보면서, 웹을 구성하는 세 가지 핵심 기술의 역할을 체감할 수 있었습니다.

## HTML — 구조를 잡는다

HTML은 페이지의 **뼈대**입니다. 헤더, 본문, 푸터 같은 영역을 나누고, 제목·문단·목록·링크 같은 콘텐츠의 의미를 정의합니다.

블로그를 만들 때도 가장 먼저 한 일은 `index.html`에 `<header>`, `<main>`, `<footer>`를 배치하는 것이었습니다. 구조가 명확해야 스타일링과 동작 추가가 수월해진다는 걸 느꼈습니다.

## CSS — 보이는 것을 다듬는다

CSS는 HTML로 잡은 구조에 **시각적 스타일**을 입힙니다. 색상, 글꼴, 여백, 레이아웃을 모두 CSS가 담당합니다.

이번에 특히 유용했던 것은 **CSS 커스텀 속성(변수)** 입니다. `--color-bg`, `--color-text` 같은 변수를 정의해 두면, 다크 모드 전환 시 변수 값만 바꾸면 전체 테마가 한꺼번에 바뀝니다.

```css
:root {
  --color-bg: #ffffff;
  --color-text: #1a1a2e;
}
.dark {
  --color-bg: #0f172a;
  --color-text: #e2e8f0;
}
```

## JavaScript — 동작을 만든다

JavaScript는 페이지에 **상호작용**을 부여합니다. 정적인 HTML을 동적으로 바꿔 주는 역할입니다.

이 블로그에서 JavaScript가 하는 일은 크게 세 가지입니다:

1. **마크다운 렌더링** — `.md` 파일을 가져와서 HTML로 변환
2. **라우팅** — URL의 해시(`#post=파일명`)를 감지해서 글 목록과 상세 화면을 전환
3. **다크 모드 토글** — 버튼 클릭 시 테마를 바꾸고 설정을 localStorage에 저장

## 세 줄 요약

| 기술 | 역할 | 비유 |
|------|------|------|
| HTML | 구조 | 건물의 골조 |
| CSS | 스타일 | 인테리어와 외관 |
| JavaScript | 동작 | 엘리베이터와 조명 스위치 |

프레임워크 없이 이 세 가지만으로도 충분히 동작하는 블로그를 만들 수 있다는 게 오늘의 가장 큰 배움이었습니다!

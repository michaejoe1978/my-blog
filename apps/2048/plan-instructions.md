# 지침: 2048 게임 계획 수립 (Plan 단계)

## 배경
이 프로젝트는 마크다운 기반 블로그 + 미니 웹앱 포트폴리오다. 프로젝트 루트의 `CLAUDE.md`를 반드시 먼저 읽고 전체 컨벤션(작업 사이클, 서브에이전트 규칙, 웹앱 규칙)을 파악하라.

## 요청받은 웹앱
- 이름: 2048
- 설명: 방향키(↑↓←→)로 숫자 타일을 밀어서 같은 숫자끼리 합치는 퍼즐 게임
- 필수 기능: 점수판(현재 점수 표시)

## 네가 할 일
`/apps/2048/` 폴더 안에 아래 내용을 담은 `spec.md` 파일을 작성하라. 코드를 작성하지 말고 계획만 세운다.

spec.md에 포함할 내용:
1. **게임 개요**: 2048 게임 규칙 요약 (4x4 그리드, 2와 4 타일 랜덤 생성, 이동/합체 로직, 승리 조건(2048 타일 생성), 패배 조건(더 이상 이동 불가))
2. **파일 구조**: `/apps/2048/` 안에 들어갈 파일 목록과 각 파일의 역할 (예: index.html, style.css, script.js)
3. **핵심 로직 설계**: 그리드 상태 표현 방식, 방향키 입력 처리, 타일 이동/합체 알고리즘 개요, 랜덤 타일 생성 타이밍
4. **UI/UX**: 점수판 표시 방식, 게임 오버/승리 화면, 모바일 대응 방식(스와이프 제스처 등 - CLAUDE.md의 "모바일에서도 사용할 수 있어야 한다" 규칙 준수)
5. **외부 라이브러리**: 사용 여부 (CLAUDE.md 규칙상 최소화, CDN은 허용되지만 이 게임은 순수 vanilla JS로 충분히 구현 가능하므로 라이브러리 없이 구현할 것을 권장)

## 제약
- 이 단계에서는 spec.md 파일만 작성한다. index.html/CSS/JS 등 실제 구현 파일은 만들지 않는다.
- `/apps/2048/` 폴더 밖의 어떤 파일도 건드리지 않는다.
- 작업이 끝나면 spec.md의 핵심 요약을 응답으로 보고하라.

# 2048 게임 - 구현 계획 (spec.md)

## 1. 게임 개요

- **그리드**: 4x4 (16칸) 정사각형 보드.
- **초기 상태**: 게임 시작 시 빈 셀 중 2곳에 랜덤 타일 생성 (2가 나올 확률 90%, 4가 나올 확률 10%).
- **이동**: 방향키(↑↓←→) 입력 시 모든 타일이 해당 방향 끝까지 밀림.
- **합체 규칙**: 이동 중 같은 숫자의 타일이 인접하게 맞닿으면 하나로 합쳐지며 값이 2배가 됨(예: 4+4=8). 한 번의 이동(턴)에서 이미 합쳐진 타일은 같은 턴 내에서 다시 합쳐지지 않음(연쇄 병합 방지).
- **턴 처리 순서**: 방향키 입력 → 이동/합체 계산 → 그리드에 변화가 있었을 경우에만 점수 갱신 및 빈 셀에 새 타일(2 또는 4) 랜덤 생성. 변화가 없으면(더 이상 그 방향으로 이동 불가) 아무 동작도 하지 않음.
- **점수**: 타일이 합쳐질 때마다 합쳐진 결과값만큼 점수에 가산 (예: 4+4→8 합체 시 +8점).
- **승리 조건**: 값이 2048인 타일이 생성되는 순간 승리. 승리 후에도 "계속하기"를 선택하면 게임을 이어갈 수 있음(더 높은 타일 도전 가능).
- **패배 조건**: 그리드가 16칸 모두 채워져 있고, 상하좌우 어느 방향으로도 이동하거나 합칠 수 있는 칸이 없을 때 게임 오버.

## 2. 파일 구조

`/apps/2048/` 폴더 안에 자체완결 구조로 구성 (블로그 루트 파일은 건드리지 않음):

| 파일 | 역할 |
|---|---|
| `index.html` | 게임 마크업. 헤더(제목), 점수판(현재 점수/최고 점수), 4x4 그리드 컨테이너, 게임오버/승리 오버레이, 새 게임 버튼, 조작법 안내 텍스트 포함. `style.css`, `script.js` 로드. |
| `style.css` | 그리드/타일 레이아웃, 타일 값별 색상 테마, 점수판 스타일, 오버레이(게임오버/승리) 스타일, 반응형(모바일) 미디어쿼리, 애니메이션(타일 등장/이동/합체 시 스케일 효과). |
| `script.js` | 게임 상태 관리, 키보드/터치 입력 처리, 이동·합체·랜덤 생성 로직, DOM 렌더링, 점수 계산 및 localStorage 최고점수 저장, 게임오버/승리 판정. |
| `spec.md` | 본 계획 문서 (이번 Plan 단계 산출물). |

- 별도 프레임워크·번들러 없이 순수 정적 파일 3개(html/css/js)로 구성해 `index.html`을 직접 열거나 정적 서빙만으로 동작.

## 3. 핵심 로직 설계

### 3.1 그리드 상태 표현
- `board`: 4x4 2차원 배열(`board[row][col]`)로 표현, 빈 칸은 `0`, 채워진 칸은 타일 숫자(2, 4, 8, ...).
- 렌더링은 상태(`board`)를 기준으로 매 턴마다 DOM을 다시 그리는 방식(단순화를 위해 매번 전체 재렌더링; 필요 시 타일 요소 재사용으로 애니메이션 개선 가능).

### 3.2 방향키 입력 처리
- `document.addEventListener('keydown', ...)`로 `ArrowUp/Down/Left/Right` 감지.
- 방향키 입력 시 `event.preventDefault()`로 페이지 스크롤 방지.
- 입력된 방향을 내부 공통 함수 `move(direction)`에 전달.
- 게임오버/애니메이션 진행 중에는 추가 입력 무시(플래그로 잠금 처리).

### 3.3 타일 이동/합체 알고리즘 개요
방향에 상관없이 "왼쪽으로 밀기" 로직 하나로 통일해 처리:
1. 방향이 `up/down/right`인 경우, 그리드를 회전(또는 행/열을 뒤집어) 항상 "왼쪽 이동" 형태로 변환.
2. 각 행에 대해:
   - **압축(compress)**: `0`이 아닌 값들을 왼쪽으로 순서 유지하며 밀착.
   - **병합(merge)**: 왼쪽부터 순회하며 인접한 두 값이 같고 아직 이번 턴에 병합되지 않았다면 합쳐서 2배 값으로 만들고, 오른쪽 칸은 `0`으로, 병합 플래그 표시(같은 타일이 한 턴에 두 번 합쳐지는 것 방지) 및 점수 누적.
   - **재압축**: 병합으로 생긴 빈칸을 다시 왼쪽으로 압축.
3. 변환했던 그리드를 원래 방향으로 역회전/역반사하여 되돌림.
4. 이동 전/후 그리드를 비교(deep compare 또는 이동 발생 플래그)해 실제 변화가 있었는지 판정.

### 3.4 랜덤 타일 생성 타이밍
- `move(direction)` 실행 결과 그리드에 변화가 있었을 때만:
  1. 빈 칸(`0`) 목록 수집.
  2. 그 중 하나를 무작위 선택, 90% 확률로 `2`, 10% 확률로 `4` 배치.
  3. 재렌더링 후 승리/패배 조건 검사.
- 초기 게임 시작 시에도 동일한 방식으로 2회 실행하여 시작 타일 2개 배치.

### 3.5 승리/패배 판정
- **승리**: 매 턴 병합 결과값 중 2048 이상이 처음 등장하면 승리 오버레이 표시(단, 이미 "계속하기"를 눌러 승리 처리를 건너뛴 상태라면 재판정하지 않음).
- **패배**: 빈 칸이 없고, 4방향 각각에 대해 가상으로 이동 가능 여부(인접 동일값 존재 여부 포함)를 검사해 모두 불가능하면 게임 오버 오버레이 표시.

## 4. UI/UX

### 4.1 점수판
- 상단에 `현재 점수` 박스와 `최고 점수(BEST)` 박스를 나란히 배치.
- 현재 점수는 매 턴 합체 시 실시간 갱신.
- 최고 점수는 `localStorage`(`best2048score` 등의 키)에 저장해 새로고침/재방문 시에도 유지, 현재 점수가 갱신될 때마다 비교 후 최고 기록 자동 갱신.

### 4.2 게임 오버 / 승리 화면
- 그리드 위에 반투명 오버레이(`position: absolute`)로 표시, 배경 딤 처리.
- **게임 오버**: "Game Over" 문구 + 최종 점수 + "다시 시작" 버튼.
- **승리**: "You Win!" 문구 + "계속하기"(오버레이 닫고 게임 지속) / "새 게임 시작" 두 가지 버튼 제공.
- "새 게임" 버튼은 상단 점수판 옆에도 상시 노출(오버레이 없이도 언제든 재시작 가능).

### 4.3 모바일 대응
- CLAUDE.md의 "모바일에서도 사용할 수 있어야 한다" 규칙 준수를 위해 스와이프 제스처 지원:
  - `touchstart`에서 시작 좌표 기록, `touchend`에서 종료 좌표와 비교.
  - 좌표 차이(dx, dy)의 절댓값이 임계값(예: 30px) 이상이고, 더 큰 축을 기준으로 방향 판별(가로 우세 시 좌/우, 세로 우세 시 상/하) 후 동일한 `move(direction)` 함수 호출.
  - 스와이프 중 페이지 자체 스크롤/새로고침(pull-to-refresh) 방지를 위해 그리드 영역에 `touch-action: none;` 적용.
- 반응형 레이아웃:
  - 그리드 크기를 고정 px 대신 `min(90vw, 420px)` 형태로 뷰포트에 맞게 축소.
  - 타일 폰트 크기도 그리드 크기에 비례한 상대 단위(`vw` 또는 `clamp()`) 사용.
  - 뷰포트 메타 태그(`width=device-width, initial-scale=1.0`) 포함해 확대/축소 이슈 방지.
- 버튼(새 게임, 다시 시작, 계속하기)은 터치 타깃 크기(최소 44px 권장) 확보.

## 5. 외부 라이브러리

- **사용하지 않음 (순수 Vanilla JS/CSS)**.
- 2048 게임 로직은 배열 연산과 DOM 조작만으로 충분히 구현 가능하며, CLAUDE.md 규칙상 외부 라이브러리는 최소화가 원칙이므로 프레임워크나 CDN 라이브러리 없이 구현한다.
- CDN을 통한 폰트(예: Google Fonts)는 선택 사항으로 허용 가능하나, 필수는 아니며 시스템 폰트(`-apple-system, "Segoe UI", Roboto` 등 루트 블로그와 동일한 폰트 스택)로도 충분.

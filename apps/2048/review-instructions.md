- `apps\2048\build-instructions.md` — Build 단계 지침(구현 요구사항 상세)

## 네가 할 일

### 1. 코드 검증
`index.html`, `style.css`, `script.js`를 꼼꼼히 읽고 spec.md의 요구사항과 대조하라. 특히:
- 이동/병합 로직이 올바른가 (한 턴에 같은 타일이 두 번 합쳐지지 않는지, 압축/병합 순서가 올바른지)
- 이동이 실제로 발생했을 때만 새 타일이 생성되는지
- 승리(2048 타일)/패배(더 이상 이동 불가) 판정 로직이 정확한지
- 점수 계산과 localStorage 최고점수 저장이 올바른지
- 방향키 preventDefault, 모바일 스와이프 처리가 구현되어 있는지
- CLAUDE.md 코딩 컨벤션(camelCase, 2칸 들여쓰기, CSS 커스텀 속성) 준수 여부
- `/apps/2048/` 밖의 파일을 건드리지 않았는지

### 2. 브라우저 동작 검증
가능한 방법으로 실제 브라우저(헤드리스 포함)에서 게임이 정상 동작하는지 확인하라:
- `browse` 스킬(헤드리스 브라우저)이 사용 가능하면 이를 이용해 `apps/2048/index.html`을 열고, 방향키 입력을 시뮬레이션해 타일 이동/합체/점수 갱신이 실제로 일어나는지 확인하라.
- 콘솔 에러(JS 에러)가 없는지 확인하라.
- 헤드리스 브라우저 도구를 쓸 수 없는 상황이면, `npx live-server` 등으로 로컬 서빙 후 접근 가능한 방법을 시도하거나, 최소한 코드를 정적으로 트레이스해서 각 시나리오(이동, 병합, 승리, 패배, 새 게임, 모바일 스와이프)를 수동으로 검증하고 그 사실을 review.md에 명시하라.

### 3. 문제 발견 시 수정
버그나 요구사항 미충족 사항을 발견하면 `/apps/2048/` 폴더 안의 파일만 직접 수정해서 고쳐라. 무엇을, 왜 고쳤는지 기록하라.

### 4. review.md 작성
`D:\ParkKiSung\Temp\ClaudeCode_Exam\apps\2048\review.md`에 다음을 작성하라:
- 검증한 항목 목록과 결과 (통과/실패)
- 실제로 브라우저(또는 그에 준하는 방법)로 확인한 내용과 방법
- 발견한 문제와 수정 내역 (있다면)
- 최종 결론: 배포 가능 여부
